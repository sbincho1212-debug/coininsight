## 하드웨어 지갑이란?

### 기본 개념

간단히 말하면, 암호화폐를 저장하는 USB 같은 물리적 장치입니다.

하지만 일반 USB와는 완전히 다릅니다.

특징:
- 개인키를 오프라인에 저장
- 인터넷에 연결 안 됨
- 해킹 불가능
- 물리적 보안

비유:
- 거래소 = 은행 계좌 (은행이 관리)
- 소프트웨어 지갑 = 지갑 앱 (폰에 저장)
- 하드웨어 지갑 = 금고 (내가 직접 보관)

<span style="color: #FFD700;">내 자산의 열쇠를 내가 직접 관리</span>

이게 핵심입니다.

### 왜 필요한가?

거래소의 위험:

문제점:
1. 해킹 위험
   - 거래소는 해커의 주요 타깃
   - 대량의 코인 보관
   - 한 번에 수천억 원 피해

2. 파산 위험
   - FTX처럼 갑자기 망함
   - 내 코인 못 찾음
   - 법적 분쟁, 오래 걸림

3. 출금 제한
   - 거래소 마음대로 출금 제한
   - "시스템 점검"이라며 차단
   - 급할 때 못 꺼냄

4. 내 것이 아님
   - "Not your keys, not your coins"
   - 거래소가 개인키 보유
   - 실질적으론 거래소 소유

실제 사례:
- Mt. Gox (2014): 85만 BTC 손실
- Coincheck (2018): 5억 달러 해킹
- FTX (2022): 80억 달러 증발
- 기타 수십 건...

<span style="color: #FFD700;">큰돈이라면 하드웨어 지갑은 필수</span>

### 어떻게 작동하나?

기술적 원리:

핵심 구조:
1. 개인키 생성 (지갑 내부)
2. 개인키 저장 (오프라인)
3. 거래 서명 (지갑 내부에서만)
4. 서명된 거래 전송 (온라인)

프로세스:
- PC/스마트폰에서 거래 생성
- 하드웨어 지갑에 전송
- 지갑 내부에서 개인키로 서명
- 서명된 거래만 PC로 전송
- 블록체인에 브로드캐스트

개인키는 절대 지갑 밖으로 안 나감!

보안 이유:
- 해커가 PC 감염해도 무용지물
- 개인키는 하드웨어 지갑 안에만
- 물리적 접근 없이는 불가능

<span style="color: #FFD700;">오프라인 보관 = 해킹 불가</span>

## 주요 하드웨어 지갑 비교

### 1. Ledger (렉저)

가장 유명한 브랜드입니다.

제품 라인업:

1. Ledger Nano S Plus
   - 가격: 약 $79
   - 크기: 작고 휴대 편함
   - 지원 코인: 5,500개 이상
   - 화면: 작은 OLED
   - 배터리: 없음 (USB 연결)

2. Ledger Nano X
   - 가격: 약 $149
   - 크기: Nano S보다 큼
   - 지원 코인: 5,500개 이상
   - 화면: 큰 OLED
   - 배터리: 있음 (블루투스)

장점:
- 세련된 디자인
- 많은 코인 지원
- Ledger Live 앱 편리
- 블루투스 (Nano X)

단점:
- 과거 고객 정보 유출 (2020)
- 클로즈드 소스 (일부)
- 블루투스 보안 우려 (일부 유저)

추천 대상:
- 초보자
- 다양한 코인 보유자
- 모바일 사용 선호 (Nano X)

## 하드웨어 지갑 설정하기

### 구매 시 주의사항

절대 중요:

안전한 구매:
1. 공식 웹사이트에서만 구매
   - ledger.com
   - trezor.io
   - safepal.com

2. 공식 리셀러 확인
   - 아마존: 공식 스토어만
   - 쿠팡, 11번가: 주의 필요

3. 중고 절대 금지
   - 이미 세팅된 지갑 위험
   - 개인키 유출 가능
   - 조작된 펌웨어 위험

위험 신호:
- 봉인 뜯어진 제품
- 이미 복구 시드 포함
- 비정상적으로 싼 가격
- 개인 판매자

<span style="color: #FFD700;">공식 채널이 아니면 절대 구매 금지!</span>

### 초기 설정 (Ledger 예시)

단계별 가이드:

1단계: 개봉 및 연결
- 박스 개봉
- USB 케이블 연결
- 전원 켜짐 확인

2단계: PIN 설정
- "Set up as new device" 선택
- 4-8자리 PIN 입력
- PIN 재확인
- 절대 잊지 말 것!

3단계: 복구 시드 생성
- 24개 단어 생성
- 순서대로 종이에 기록
- 철저히 확인
- 안전한 곳에 보관

4단계: 복구 시드 확인
- 지갑이 무작위로 단어 질문
- 정확히 입력
- 검증 완료

5단계: 앱 설치
- Ledger Live 다운로드
- PC/스마트폰 설치
- 지갑 연결
- 앱 업데이트

6단계: 코인 앱 설치
- Ledger Live에서 설치
- Bitcoin, Ethereum 등
- 필요한 코인만

완료!

### 복구 시드의 중요성

생명줄입니다.

복구 시드란?
- 24개 영어 단어
- 개인키의 마스터 키
- 지갑 복구 유일한 방법

시나리오:
- 지갑 분실/파손?
- PIN 잊어버림?
- 새 지갑 구매?

→ 복구 시드로 모든 자산 복구 가능

반대로:
- 복구 시드 분실?
- → 자산 영영 잃음
- → 복구 불가능

<span style="color: #FFD700;">복구 시드 = 내 전 재산</span>

보관 방법:

절대 하면 안 되는 것:
- 디지털 저장 (사진, 파일)
- 클라우드 업로드
- 이메일 전송
- 메신저 공유
- PC/폰 메모

해야 하는 것:
- 종이에 손으로 기록
- 금속판에 각인 (화재 대비)
- 안전한 장소 보관 (금고)
- 여러 곳 분산 보관
- 가족에게 위치 알림 (비상시)

추천 제품:
- Cryptosteel (금속 백업)
- Billfodl (화재/수해 방지)
- 또는 스테인리스 직접 각인

## 하드웨어 지갑 사용법

### 코인 입금하기

간단합니다.

절차:
1. Ledger Live 열기
2. 받으려는 코인 선택 (예: Bitcoin)
3. "Receive" 클릭
4. 하드웨어 지갑에서 확인
5. 주소 복사
6. 거래소/다른 지갑에서 전송

중요:
- 반드시 하드웨어 지갑 화면에서 주소 확인
- PC 화면만 믿지 말 것
- 악성코드가 주소 바꿀 수 있음

첫 전송:
- 소액으로 테스트
- 제대로 도착 확인
- 그 다음 본 금액

<span style="color: #FFD700;">첫 전송은 반드시 소액 테스트!</span>

### 코인 출금하기

조금 더 신중하게:

절차:
1. Ledger Live 열기
2. 보낼 코인 선택
3. "Send" 클릭
4. 받는 주소 입력
5. 금액 입력
6. 수수료 확인
7. 하드웨어 지갑에서 거래 확인
   - 주소 확인
   - 금액 확인
   - 버튼 눌러 승인
8. 전송 완료

주의사항:
- 주소 한 글자라도 틀리면 영영 잃음
- 복사-붙여넣기 후 재확인
- 하드웨어 지갑 화면과 대조
- 수수료 너무 낮으면 느림

### DeFi 연결 (메타마스크)

고급 기능:

하드웨어 지갑으로 DeFi 안전하게 사용:

설정:
1. MetaMask 설치
2. "Connect Hardware Wallet" 선택
3. Ledger 연결
4. 계정 선택
5. 완료

사용:
- DeFi 프로토콜 접속 (Uniswap 등)
- 거래 실행
- MetaMask 팝업
- 하드웨어 지갑에서 승인
- 거래 완료

장점:
- DeFi 편리하게 사용
- 개인키는 안전하게 보관
- 해킹 걱정 없음

주의:
- 피싱 사이트 조심
- 승인 내용 꼼꼼히 확인
- 이상하면 거부

## 절대 하면 안 되는 실수

### 1. 복구 시드 디지털 보관

최악의 실수:

하면 안 되는 것:
- 사진 찍기
- 클라우드 저장
- 이메일 전송
- 카카오톡 메시지
- 에버노트/노션 저장

이유:
- 해킹 당하면 끝
- 클라우드 유출 사례 많음
- 메신저 해킹 흔함
- 폰 분실 시 노출

결과:
- 복구 시드 탈취
- 전 재산 도난
- 복구 불가

<span style="color: #FFD700;">디지털 저장 = 자산 포기</span>

### 2. 복구 시드 온라인 입력

피싱 공격:

수법:
- "지갑 동기화 필요합니다"
- 가짜 웹사이트
- 복구 시드 입력 요구
- → 입력하면 바로 도난

진실:
- Ledger/Trezor는 절대 복구 시드 요구 안 함
- 웹사이트에 입력할 일 없음
- 이메일로 문의도 안 함

대응:
- 의심되면 무시
- 공식 사이트만 방문
- URL 철저히 확인

### 3. PIN 여러 번 틀리기

자동 초기화:

위험:
- Ledger: 3번 틀리면 초기화
- Trezor: 16번까지 허용하지만 시간 지연

결과:
- 지갑 초기화
- 복구 시드 없으면 자산 잃음

대비:
- PIN 안전한 곳에 메모
- 또는 확실히 기억
- 애매하면 복구 시드로 복구 후 새 PIN

### 4. 펌웨어 비공식 업데이트

조작된 펌웨어:

위험:
- 중고 지갑 구매 시
- 이미 조작된 펌웨어
- 개인키 유출 코드

예방:
- 새 제품만 구매
- 첫 사용 시 펌웨어 재설치
- 공식 앱에서만 업데이트

### 5. 복구 시드 한 곳만 보관

화재, 도난, 분실:

문제:
- 집에만 보관
- 화재로 소실
- 또는 도난
- → 복구 불가

해결:
- 2-3곳 분산 보관
- 다른 장소 (집, 부모님 집, 금고)
- 금속판 사용 (화재 대비)

주의:
- 한 곳에 전부 보관 위험
- 그렇다고 너무 많은 곳도 위험
- 2-3곳이 적당

페이퍼 지갑:
- 보안: 높음

- 비용: 무료
- 추천 용도: 장기 보관

### 언제 하드웨어 지갑을 써야 할까?

하드웨어 지갑 권장:

- 편의성 중요

- 이동 중 거래

조합 전략:
- 거래소: 거래용 소액 (10%)

## 고급 팁

### 1. 멀티시그 (Multisignature)

더 강한 보안:

개념:
- 여러 개 지갑 필요
- 예: 3개 중 2개 서명 필요
- 한 개 잃어도 안전

설정:
- Electrum (비트코인)
- Gnosis Safe (이더리움)
- 여러 하드웨어 지갑 조합

장점:
- 극강 보안
- 한 지갑 도난/분실 OK
- 상속 계획 가능

단점:
- 복잡함
- 여러 기기 필요
- 초보자에게 어려움

추천 대상:
- 초고액 보유자
- 기관 투자자
- 상속 계획 필요자

### 2. 패스프레이즈 (25번째 단어)

추가 보안층:

기능:
- 24 단어 + 패스프레이즈
- 선택 사항
- 다른 지갑 생성

활용:
- 24 단어: 소액 지갑 (미끼)
- 24 단어 + 패스프레이즈: 진짜 자산

장점:
- 강탈 대비
- 복구 시드 탈취 돼도 안전
- 여러 지갑 생성 가능

단점:
- 패스프레이즈 잊으면 복구 불가
- 관리 복잡
- 실수 위험

주의:
- 패스프레이즈도 안전하게 보관
- 하지만 복구 시드와 분리
- 둘 다 잃으면 끝

### 3. 정기 점검

건강검진처럼:

체크리스트:
- 분기별 1회

확인 사항:
- 지갑 정상 작동?
- 펌웨어 업데이트?
- 복구 시드 안전한가?
- PIN 기억하는가?
- 백업 확인 (소액 복구 테스트)

이유:
- 막상 필요할 때 고장 발견 늦음
- 미리 점검하면 안심
- 문제 조기 발견
